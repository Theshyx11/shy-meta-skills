# shy-theory_grounding

## 目标

把“我遇到的这个具体困境”当成一个理论检索任务，逆向寻找人类知识史里真正研究过相似结构的人物、理论、论文、经典著作和学科传统。

本 skill 不是预制理论路由表。它必须回答：

```text
这个问题历史上谁研究过？
它在学术/思想史/技术史里叫什么？
它属于哪些学科和理论传统？
有哪些人物、著作、论文或关键词能追溯？
哪些理论和我的问题是强同构，哪些只是弱类比？
理论如何改写我现在的提问方式？
下一步应该怎样验证这些理论是否真的帮到了当前问题？
```

## 和相邻 skill 的边界

```text
shy-knowledge_topology：从概念到知识结构。
shy-theory_grounding：从具体困境到开放理论检索。
shy-kill_unknow_unknowns：从鬼打墙到问题审判和破局实验。
```

如果用户要“概念和哪些概念并列、属于哪个集合”，用 `shy-knowledge_topology`。
如果用户要“我是不是问错了、怎样跳出鬼打墙”，用 `shy-kill_unknow_unknowns`。
如果用户要“历史上谁研究过、有哪些理论人物、理论源流、论文和学科传统”，用本 skill。

## 核心原则

1. 不要使用预制理论矩阵作为答案来源。
2. 先把用户问题抽象成可检索的失败结构、因果结构或认知结构。
3. 再生成中英文检索词、同义词、上位词、相邻学科词。
4. 能联网时必须检索；不能联网时必须明确标注“未检索验证”，只能给检索计划和候选方向。
5. 每个理论候选都要说明来源线索、匹配强度、同构点和不匹配点。
6. 允许结论是“没有找到强理论匹配”，不能硬塞名人和理论。
7. 预制理论清单只能作为最后的盲区补查清单，不能替代开放检索。

## 工作流

### 1. 抽象用户困境

先把现实问题压缩成 2 层：

```text
现实困境：{用户的具体场景}
抽象问题结构：{更底层的失败类型、因果结构、决策结构或认知结构}
```

常见抽象结构示例：

```text
代理指标误导真实目标
反复调参但变量没有机制解释
异常样本持续堆积
问题定义随行动改变
局部优化陷阱
搜索空间错误
复杂系统反馈延迟
概念混用导致伪问题
目标、约束、行动混在一起
理论不可证伪
旧解释只能靠补丁维持
```

这些只是抽象示例，不是理论答案。

### 2. 生成检索词

为每个抽象结构生成检索词：

```text
中文关键词：{...}
英文关键词：{...}
相邻学科关键词：{...}
反向关键词：{用于排除表面相似理论的关键词}
```

检索词必须包含：

- 用户原始领域词，例如量化策略、盘口、变量、假设检验。
- 抽象结构词，例如 proxy variable、degenerating research programme、model misspecification。
- 学科入口词，例如 philosophy of science、statistical learning、market microstructure。

### 3. 开放检索

能使用互联网时，必须优先检索以下来源：

```text
Google Scholar / Semantic Scholar / arXiv / SSRN
Stanford Encyclopedia of Philosophy / Internet Encyclopedia of Philosophy
大学课程讲义或出版社页面
论文原文、作者主页、经典著作介绍
百科页面只能作为入口，不能作为最终依据
```

检索时至少覆盖：

```text
学术关键词检索
人物/理论名检索
相邻学科检索
反例检索：搜索为什么该理论可能不适用
```

如果当前环境不能联网：

```text
明确写：当前未做外部检索验证。
输出检索计划，而不是伪装成已检索结论。
把所有理论标成“待检索候选”。
```

### 4. 筛选理论候选

每个理论候选必须过 5 个问题：

```text
1. 它原本研究的问题是什么？
2. 它是否处理了用户困境的底层结构，而不只是表面相似？
3. 它能不能改写用户当前的问题定义？
4. 它能不能生成新的观察方式或验证动作？
5. 有没有明确人物、著作、论文、理论传统或检索来源可追溯？
```

匹配强度分为：

```text
强匹配：原始问题结构和用户困境高度同构，可以直接改写问题。
中匹配：提供观察工具或验证框架，但不能完全覆盖问题。
弱匹配：只是启发或类比，不能作为主要理论地基。
不匹配：表面相似但底层结构不同，必须剔除。
```

### 5. 输出理论矩阵

输出 5-8 个最相关理论，不要百科式堆叠。格式：

```text
| 匹配强度 | 理论/人物 | 来源线索 | 原始问题 | 和当前困境的同构点 | 不匹配点/风险 | 如何改写问题 |
```

`来源线索` 可以是经典著作、论文名、作者、学派、百科入口或需要进一步检索确认的关键词。不要伪造页码、论文细节或不存在的引用。

### 6. 生成理论改写

对强匹配和中匹配理论分别输出：

```text
如果用 {理论} 看：
旧问题：{用户原来的问法}
新问题：{被理论改写后的问法}
新增观察变量：{理论要求你看什么}
新增验证动作：{如何判断这个理论是否真的更好}
```

### 7. 盲区补查

最后才允许使用内置盲区清单做补查。它不是答案来源，只用于问：

```text
是否遗漏了语言哲学、科学哲学、统计学习、系统论、复杂性理论、决策理论、市场微观结构、组织学习、设计理论等相邻传统？
```

如果补查发现候选理论，必须回到第 3 步继续检索或标注为“待检索候选”。

## 输出模板

```markdown
**问题抽象**
现实困境：{...}
抽象问题结构：{...}

**检索设计**
中文关键词：{...}
英文关键词：{...}
相邻学科关键词：{...}
反向排除关键词：{...}

**检索状态**
{已检索的来源 / 或 当前未做外部检索验证}

**理论候选矩阵**
| 匹配强度 | 理论/人物 | 来源线索 | 原始问题 | 同构点 | 不匹配点/风险 | 如何改写问题 |
|---|---|---|---|---|---|---|

**理论改写**
1. 用 {理论} 看：
   - 旧问题：{...}
   - 新问题：{...}
   - 新增观察变量：{...}
   - 最小验证动作：{...}

**盲区补查**
- 可能遗漏的相邻理论传统：{...}
- 需要继续检索的关键词：{...}

**一句话**
{把理论地基压缩成一句可记住的话}
```

## 禁止事项

- 不要把预制理论矩阵当答案。
- 不要只列名人和书单。
- 不要把表面相似当理论同构。
- 不要用理论替代现实验证。
- 不要伪造论文、页码、引用。
- 不要把所有理论都说成同等重要。
- 不要默认用户当前的问题定义是对的。
- 不要在没有检索时假装已经检索。

## 质量标准

合格输出必须让用户得到：

```text
我这个困境被抽象成了什么可检索结构。
哪些理论是检索出来的，哪些只是待验证候选。
哪些人物和理论传统是真正强同构。
每个理论如何改写我的问题。
下一步用什么关键词和最小实验验证理论是否有用。
```
