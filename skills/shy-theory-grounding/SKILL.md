# shy-theory_grounding

## 目标

把“我遇到的这个困境”逆向定位到人类知识史里已经出现过的理论地基。

本 skill 不负责直接解决问题，而是回答：

```text
这个问题历史上谁研究过？
它在学术/思想史里叫什么？
它属于哪个学科的哪类问题？
哪些理论能改写我现在的提问方式？
我应该从哪些人物、著作、关键词开始补理论？
```

## 和相邻 skill 的边界

```text
shy-knowledge_topology：从概念到知识结构。
shy-theory_grounding：从困境到理论地基。
shy-kill_unknow_unknowns：从鬼打墙到问题审判和破局实验。
```

如果用户要“概念和哪些概念并列、属于哪个集合”，用 `shy-knowledge_topology`。
如果用户要“我是不是问错了、怎样跳出鬼打墙”，用 `shy-kill_unknow_unknowns`。
如果用户要“历史上谁研究过、有哪些理论人物、理论源流是什么”，用本 skill。

## 核心方法

不要从关键词硬搜理论。先把用户困境抽象成失败类型，再定位学科和理论。

```text
现实困境
→ 失败类型抽象
→ 学科归属
→ 理论人物/经典理论
→ 理论如何改写问题
→ 可检索关键词/阅读入口
→ 最小验证动作
```

筛选理论的标准：

1. 它是否处理了用户困境的底层结构，而不只是表面相似。
2. 它是否能改写用户当前的问题定义。
3. 它是否能生成新的行动方式、观察方式或验证方式。
4. 它是否有明确人物、著作、论文或理论传统可追溯。

## 理论定位矩阵

按困境选择，不要全部堆上去。

| 困境结构 | 学科/传统 | 代表人物/理论 | 适用方式 |
|---|---|---|---|
| 问题可能是语言制造的 | 语言哲学 | 维特根斯坦，《逻辑哲学论》《哲学研究》 | 审词义、审使用场景、识别伪问题 |
| 反复修动作但不突破 | 组织学习 | Chris Argyris & Donald Schon，双环学习 | 从改行动升级到改目标、规则、假设 |
| 问题定义会随行动改变 | 规划理论 | Rittel & Webber，Wicked Problems | 承认定义问题本身就是问题的一部分 |
| 实践中难的是设题 | 设计/反思性实践 | Donald Schon，Problem Setting | 在行动中重构“这到底是什么问题” |
| 卡在同一套搜索空间 | 认知科学/AI | Newell & Simon，Problem Space | 审初始状态、目标状态、操作符 |
| 看不见新用法 | 格式塔心理学 | Karl Duncker，功能固着 | 识别惯常用途如何锁死思路 |
| 不是单点问题，而是一团关系 | 系统论 | Russell Ackoff，Messes；Peter Checkland，SSM | 从 solve 改为 dissolve，重组关系 |
| 异常堆积，旧解释越来越勉强 | 科学哲学 | Thomas Kuhn，范式转换 | 判断是否进入旧范式危机 |
| 需要从异常生成新假说 | 实用主义/逻辑学 | C. S. Peirce，Abduction | 从异常反推可能机制 |
| 因果域不清，不同问题要不同打法 | 复杂性理论 | Dave Snowden，Cynefin | 判断 simple/complicated/complex/chaotic |
| 改善 A 恶化 B | 发明方法论 | Altshuller，TRIZ | 把卡点表述成技术/业务矛盾 |
| 理论不可证伪 | 科学哲学 | Karl Popper，可证伪性 | 把说法改成可被事实打脸的命题 |
| 研究方向只剩补丁 | 科学哲学 | Imre Lakatos，研究纲领 | 判断是进步问题转换还是退化问题转换 |
| 人在不确定中行动 | 实用主义 | John Dewey，Inquiry | 把问题看成调查过程，而非一次性答案 |
| 决策受框架影响 | 行为经济学 | Kahneman & Tversky，框架效应/前景理论 | 审当前表述如何改变选择 |
| 系统由反馈回路主导 | 系统动力学 | Jay Forrester / Donella Meadows | 找反馈、延迟、杠杆点 |
| 选择不是最优，而是满意 | 决策理论 | Herbert Simon，有限理性/满意化 | 降低“最优解”执念，改找可行满意解 |

## 工作流

### 1. 抽象用户困境

先用一句话写出：

```text
你的现实困境是：{具体场景}
抽象后的失败类型是：{更底层结构}
```

失败类型示例：

```text
问题定义不稳定
代理指标误导真实目标
局部优化陷阱
搜索空间错误
复杂系统反馈延迟
异常样本无法解释
概念混用导致伪问题
目标、约束、行动混在一起
```

### 2. 定位学科归属

给出 2-4 个候选学科/理论传统，并说明为什么：

```text
这不是单纯的商业问题，它至少同时属于：
1. {学科}：因为 {理由}
2. {学科}：因为 {理由}
```

### 3. 找理论人物

输出 5-8 个最相关理论，不要百科式堆叠。格式：

```text
| 理论/人物 | 经典来源 | 原始问题 | 和你当前困境的同构点 | 它如何改写你的问题 |
```

“同构点”必须具体，不能只写“都和问题有关”。

### 4. 排优先级

把理论分成三层：

```text
第一优先级：直接改写问题定义。
第二优先级：提供观察/验证工具。
第三优先级：补背景知识，暂时不急。
```

### 5. 生成理论改写

对前 3 个理论分别输出：

```text
如果用 {理论} 看，你的问题不再是：
{旧问题}

而是：
{新问题}
```

### 6. 给检索和阅读入口

提供中英文关键词，不要伪造具体论文细节。对不确定的文献，标注需要检索确认。

```text
中文关键词：{...}
英文关键词：{...}
经典入口：{人物 + 著作/理论名}
```

### 7. 最小验证动作

理论不能停在阅读。每个优先理论给一个最小验证动作：

```text
用这个理论，今天可以做的一个动作是：{动作}
它验证的是：{哪个问题定义是否更好}
```

## 输出模板

```markdown
**困境抽象**
现实困境：{...}
失败类型：{...}

**学科定位**
1. {学科}：{原因}

**理论人物矩阵**
| 优先级 | 理论/人物 | 经典来源 | 原始问题 | 同构点 | 如何改写你的问题 |
|---|---|---|---|---|---|

**三种理论改写**
1. 用 {理论} 看：
   - 旧问题：{...}
   - 新问题：{...}

**检索入口**
- 中文关键词：{...}
- 英文关键词：{...}
- 经典入口：{...}

**最小验证动作**
1. {动作}
   - 验证：{...}
   - 如果成立：{...}
   - 如果不成立：{...}

**一句话**
{把理论地基压缩成一句可记住的话}
```

## 禁止事项

- 不要只列名人和书单。
- 不要把表面相似当理论同构。
- 不要用理论替代现实验证。
- 不要伪造论文、页码、引用。
- 不要把所有理论都说成同等重要。
- 不要默认用户当前的问题定义是对的。

## 质量标准

合格输出必须让用户得到：

```text
原来我这个困境属于哪些历史问题。
哪些人已经研究过类似结构。
每个理论会怎样改变我现在的提问方式。
我该从哪几个关键词和最小实验开始。
```
