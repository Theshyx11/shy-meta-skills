# shy-language-precision-auditor

## 目标

把用户的低分辨率表达改造成更专业、更可执行、更可验证的问题语言。

本 skill 只在被明确调用或用户要求改善表达质量时使用。不要把它作为每次回答的默认前置流程。

## 使用边界

使用本 skill：

- 用户明确说“审计语言”“语言贫瘠”“字段池”“问题重写”“把这个说专业”。
- 用户想把模糊问题改成专业问题、研究问题、执行问题或决策问题。
- 用户想补变量、字段、约束、反例、判断标准和可验证口径。
- 用户连续使用“怎么做”“优化”“提升”“感觉不对”“有没有机会”等低分辨率表达，并且本轮目标是改善表达或问题定义。

不要使用本 skill：

- 用户只要直接执行命令、修改文件、写代码或给结论。
- 用户没有要求审计表达，且继续做正事更重要。
- 其他领域 skill 已经能直接解决问题，不需要额外语言审计。

## 审计原则

- 审计对象是语言、概念、字段池、问题框架和表达精度，不审判人格。
- 语气可以锋利，但必须服务于扩充可思考空间。
- 审计必须领域化：量化交易问题补市场微观结构、统计检验、执行成本和风险暴露；商业问题补供需、定价、渠道、转化、信任、组织和现金流；个人决策问题补约束、偏好、机会成本、情绪噪音和长期状态。
- 涉及代码、字段、变量、指标或配置名时，每一个都必须用中文解释。
- 简单问题压缩输出，复杂问题完整输出。

## 输出结构

默认按四段输出：

1. **语言审计**：指出当前表达哪里虚、扁、缺变量、断因果、缺时间尺度、边界含混、缺反例或约束。
2. **字段池扩充**：列出至少 8 个关键字段，每个字段解释中文含义、为什么重要、改变哪个判断。
3. **句子升级**：给出 3-5 句用户可以直接使用的高密度表达。
4. **问题重写**：把原问题改写成更专业、更可执行、更可验证的问题。

如用户还需要继续解决问题，在完成以上四段后再给出正式回答；否则停在问题重写即可。

## 字段解释模板

```text
字段名：
中文含义：
为什么重要：
它改变的判断：
常见误用：
```

## 质量标准

一次合格审计必须让用户看到：

- 原话为什么无法指导行动。
- 缺的不是“更努力”，而是哪些具体字段、变量、约束和判断口径。
- 改写后的问题如何更容易被研究、验证或执行。
