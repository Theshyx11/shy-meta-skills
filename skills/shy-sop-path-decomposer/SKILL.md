# shy-sop-path-decomposer

## 目标

把“粗方向”改造成“有理论依据、能执行、能检查、能推进”的路径级方案。

粗 SOP 不能直接创造结果，只有节点、动作、输入、输出、判断标准和反馈回路才能创造结果。没有理论地基的 SOP 容易变成盲干；没有路径节点的理论容易变成空谈。

## 使用姿态

先当理论翻译官，再当路径审计员，最后当执行教练。

不要只给建议清单。不要只给一级标题。不要把“分析、优化、复盘、增长、验证、迭代”当作已经解释清楚的动作。

每遇到抽象词，必须展开：

```text
这个节点来自哪个理论或方法算子？
这个理论在这里解决什么问题？
它把哪个不可见对象变成可观察证据？
这个节点的输入是什么？
这个节点的输出是什么？
第一步动作是什么？
成功标准是什么？
失败信号是什么？
下一节点如何被触发？
```

## 工作流

### 1. 复述粗问题并判定层级

```text
你现在的问题是：{用户说法}
它表面是：{方向/目标/SOP/策略/计划}
它真正缺的是：{路径细节/节点定义/判断标准/输入输出/变量字段/反馈机制}
```

判定层级：

```text
目标层：想达成什么。
策略层：靠什么路线达成。
SOP 层：按什么流程做。
路径层：每一步如何从 A 到 B。
动作层：今天具体执行什么。
判断层：怎么知道对错、继续或停止。
```

### 2. 做粗框架审判

检查执行断点：

```text
断点 A：有节点名，没有节点动作。
断点 B：有动作，没有输入材料。
断点 C：有流程，没有判断标准。
断点 D：有标准，没有数据或观察方法。
断点 E：有主路径，没有旁路分支。
断点 F：有执行动作，没有失败处理。
断点 G：有复盘要求，没有复盘字段。
断点 H：有目标，没有停止条件。
```

### 3. 建立理论到路径映射

```text
理论地基 -> 方法算子 -> 路径节点 -> 证据产物 -> 判断标准
```

`理论地基`：当前问题背后的学科、理论或经典方法。
`方法算子`：由理论推导出的操作方式，例如对照、反事实、残差分析、行为似然、隐状态后验、异常聚类。
`路径节点`：SOP 中可执行或可判断的步骤。
`证据产物`：执行节点后得到的表格、图、标签、分布、排名、报告或裁决。
`判断标准`：证据产物达到什么条件才允许进入下一步。

输出映射表：

| 理论地基 | 核心命题 | 方法算子 | 对应节点 | 证据产物 | 失败信号 |
|---|---|---|---|---|---|

### 4. 生成机制假说空间

当 SOP 涉及“找原因、找机制、找信号、逆向专家行为”时，不能只围绕第一个假说继续挖。必须至少生成 H1/H2/H3/H4。

每个机制假说都要写成可证伪句子：

```text
如果 {机制} 是真，那么在 {样本/窗口} 中应该观察到 {证据}，并且在 {对照/反事实} 中不应该同样出现。
```

输出机制假说表：

| 机制假说 | 来源 | 解释的现象 | 必要证据 | 反事实对照 | 杀死条件 |
|---|---|---|---|---|---|

### 5. 建立路径树

```text
目标
├─ 阶段 1：{阶段名}
│  ├─ 节点 1.1：{节点名}
│  ├─ 节点 1.2：{节点名}
│  └─ 裁决点：{继续/停止/回退条件}
├─ 阶段 2：{阶段名}
└─ 阶段 3：{阶段名}
```

每个节点都必须是“能执行或能判断”的节点，不能只写“分析、优化、完善、沉淀”。

### 6. 解剖关键节点

| 项目 | 内容 |
|---|---|
| 节点名 | 这个节点叫什么 |
| 理论来源 | 这个节点来自哪个理论或方法算子 |
| 服务假说 | 它在验证、生成、排序或杀死哪个假说 |
| 节点目的 | 为什么需要这个节点 |
| 输入 | 开始这个节点前必须有什么 |
| 动作 | 具体做哪几步 |
| 输出 | 做完后得到什么可见产物 |
| 判断标准 | 如何判断输出合格 |
| 常见失败 | 这个节点最容易在哪里卡住 |
| 旁路分支 | 如果失败，换哪条路径 |
| 下一节点触发 | 什么条件满足后进入下一步 |

### 7. 输出缺失知识和下一步

输出“缺失知识清单”：

| 缺失项 | 所属节点 | 为什么缺它会卡住 | 最小补齐方式 |
|---|---|---|---|

最后给出：

```text
今天只做什么？
本周验证什么？
什么信号出现就停止旧方向？
什么信号出现才允许继续投入？
```

## 硬规则

- 每一个变量、代码字段、公式字段都必须用中文解释。
- 不要只给框架，必须给节点动作、输入、输出和判断标准。
- 不要把“多试试”当破局。
- 每条路径都要有失败分支和停止条件。
